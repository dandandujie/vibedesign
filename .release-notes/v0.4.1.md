## v0.4.1

安全与健壮性补丁：一轮 Codex 全项目源码审查发现 14 个问题，逐项修复并验证（server/web `tsc`、`build`、真窗口冒烟）。核心编辑框架未改动。

### 重要修复
- **打包依赖修复**：v0.4.0 的桌面安装包可能因内嵌服务启动时无法解析 `playwright` / `ffmpeg-static` 而**打不开**；现将这些运行时依赖提升到根依赖并纳入打包清单，安装包可正常启动。
  - 注：像素导出 / 视频渲染还依赖 Chromium 浏览器二进制，若安装包内暂缺该二进制，这两个功能会在点击时报错——修复应用启动是本版重点。
- **本地 API 加固**：服务仅监听 `127.0.0.1`，Host / Origin / CORS 收窄到本地来源；已保存的 API Key 仅在同一 Provider 且 Base URL 一致时才复用掩码，杜绝被恶意网页借掩码窃取。
- **iframe 消息加固**：画布与 inspector 校验 postMessage 来源，忽略非本设计窗口的消息（保留同源，编辑 / 标注功能不受影响）。

### 其它修复
- Live Artifacts：`http_json` 刷新改为逐跳复验重定向（防 SSRF 重定向绕过）；映射目标路径拒绝 `__proto__` 等（防原型污染）；sidecar 路径严格校验。
- 渲染接口：加 HTML 大小上限、并发上限、超时与客户端断开取消，PNG 像素上限。
- 数据持久化：改为原子写入（临时文件 + rename），解析失败时备份损坏文件而非静默清空——避免异常退出导致数据丢失。
- 自动保存：离开页面 / 关闭前立即保存，保存失败会明确提示。
- 其它：SSE 兼容 CRLF 流 + 缓冲上限；绘制撤销顺序修正（画完可正确撤销）；视频渲染临时目录异常时也清理；Electron 外链精确校验后再打开；透明度可正确设为 0。
