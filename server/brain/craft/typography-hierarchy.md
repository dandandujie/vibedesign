# Typography hierarchy

How to make a layout's type feel *authored* rather than assembled. Requires the
base `typography` rules; this layer is about relationships between levels.

## One clear entry point
Every screen has exactly one element the eye lands on first — usually the H1 or
the hero line. Make its jump from the next level down decisive (a real size
step plus a weight/color shift), not a timid 2px difference. If two elements
compete for "first," the hierarchy has failed.

## Contrast between levels, not just size
Distinguish levels with more than one axis at once: size **and** weight **and**
color. A muted eyebrow label (small, tracked, secondary color) over a large
bold headline over normal body reads instantly. Three sizes that are all the
same weight and color read as a list, not a hierarchy.

## Limit the number of levels
Most surfaces need 3–4 type levels total (e.g. eyebrow / headline / body /
meta). Inventing a fifth and sixth level flattens the whole thing — each extra
level dilutes the ones above it.

## Group with space, separate with space
Related items sit closer; unrelated items get a bigger gap (proximity). The
space *above* a heading should be larger than the space *below* it, so the
heading visually belongs to the content it introduces.

## Rhythm and density
Alternate dense and open passages so the page has cadence. A wall of evenly
sized, evenly spaced blocks reads as generated. One tight information section
followed by one that breathes is what makes a layout feel intentional.

## Measure and alignment
Keep headings and body on a shared left edge / grid so the eye tracks a clean
vertical line. Headlines can run wider, but body copy still respects the
60–75ch measure regardless of the container.
